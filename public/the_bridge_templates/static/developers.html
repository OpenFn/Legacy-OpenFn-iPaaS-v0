Adding your product couldn't be easier.


  *metadata layer

Describe

Learn schema

We make a call to your server, asking for a description of a particular user's forms, objects, or tables. We'll expect data to be JSON Schema compliant.

*data layer


Listen

"Inbox" receives data

If you can't do push notifications, we can set up a periodic pull. We've got background workers scheduled to call out to your DB and fetch new records.
Translate

To JSON

Once data from your app reaches us, it must be translated to our universal syntax.
Transform

Field mapping

Users define the transformation, using both "origin" and "destination" schemas.
Prepare

From JSON

You take our payload, in the universal JSON, and convert it back to whatever format your app prefers. In some cases, a single JSON document will be prepared into several distinct operationsâ€”create, update, upsert.
Push

Bon voyage

We fire the prepared message (along with secure client credentials) to your server.

A new "adapter" is typically a gem that responds to "describe", "translate" and "prepare" calls. Don't hesitate to get in touch with us if you're interested in developing an adapter for your product, or need help using any of these components in your own projects.
